# Plan — Path A : vraie intégration freebuff

## Goal

Remplacer les deux executors parallèles qui postent le mauvais wire shape par un seul `FreebuffExecutor` câblé dans `open-sse`, qui implémente le protocole 3-step `startAgentRun → chat/completions → finishAgentRun` observé dans la capture live et qui permet à `POST /api/v1/chat/completions` (modèle `freebuff/...`) de retourner un stream OpenAI authentique au lieu d'un 403.

## Constraints

- **Pas de bypass** du check `free_mode_cli_required` — on s'intègre _autour_, pas _à travers_ (audit §8).
- **Backward compat** : schema `FreebuffConnection`, PKCE login, `/quota`, `/streak`, `/session`, paste `credentials.json` restent fonctionnels.
- **Tests verts** : les 17 fichiers de tests existants ne régressent pas, sauf mise à jour explicite pour refléter le nouveau wire.
- **i18n FR** : plus aucun `[TODO i18n fr]` dans `fr.json` après Chunk 8.
- **Pas de nouvelle dépendance npm** — on reste sur `node-machine-id` (déjà présent).
- **Strict typing** : `strict: false` au global mais typage explicite sur les nouvelles fonctions (convention du repo).
- **Plan mode** : l'écriture disque (docs/research/\*.md, tests/) nécessite que tu passes en `/permissions auto` ou que tu acceptes les demandes de permission au coup par coup.

### Chunk 4 — Rewrite `chatIntegration` avec le vrai body shape

**Scope** : réécrire le dispatcher chat pour envoyer les 8 champs top-level capturés, gérer le `codebuff_metadata` spread, enchaîner START → chat → FINISH.

- **Files Changed** :
  - `src/lib/providers/freebuff/chatIntegration.ts` (REWRITE, ~280 LoC) — nouveau `routeFreebuffChat` qui : (1) charge creds via `loadFreebuffCredentials`, (2) résout agent via `getFreebuffAgentId`, (3) construit `sessionState` via nouveau module, (4) appelle `startAgentRun`, (5) POST le body complet à `/api/v1/chat/completions`, (6) pipe le stream à travers le transformer OpenAI/Anthropic, (7) `finally { finishAgentRun(...) }`
  - `src/lib/providers/freebuff/sessionState.ts` (NEW, ~80 LoC) — helper `buildSessionState({messages, fileContext?, knowledgeFiles?})` qui retourne la forme `sessionState.fileContext + sessionState.mainAgentState` du wire capturé
- **Depends On** : Chunks 1, 2, 3
- **Accept When** :
  - Le body envoyé contient **exactement** les 8 champs du wire capturé : `promptId`, `clientSessionId`, `fingerprintId`, `agentId`, `prompt`, `toolResults`, `sessionState`, `extraCodebuffMetadata`
  - Les champs de `codebuff_metadata` sont promus top-level conformément à la capture Chunk 1
  - `startAgentRun` est appelé **avant** le POST chat ; `finishAgentRun` est appelé dans un `finally` (même si le stream est interrompu par un abort)
  - Le response body reste un SSE stream compatible OpenAI (output inchangé de `createOpenAITransformer`)
  - `routeFreebuffChat` reste signature-compatible avec l'ancien (mêmes params, même `Promise<Response>`)
- **Test Coverage** : `tests/unit/lib/providers/freebuff-chat-integration.test.ts` (NEW) — golden test sur body shape avec snapshot du JSON envoyé ; test du lifecycle (START appelé, FINISH appelé même en cas d'abort)
- **Open Questions** :
